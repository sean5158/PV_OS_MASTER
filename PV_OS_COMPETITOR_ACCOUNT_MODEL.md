## 一、竞品账号库定位

### 1.1 核心定位

**竞品账号库不是为了分析竞品销售。**

竞品账号库的唯一商业目的：

```
竞品账号库
    │
    ▼
comment_collector_agent 读取账号列表
    │
    ▼
采集这些账号的视频评论区
    │
    ▼
从中发现真实潜在客户的评论
    │
    ▼
进入 PV_OS 客户发现链路
```

| 不是 | 而是 |
|------|------|
| ❌ 研究竞品定价策略 | ✅ 找到竞品评论区里的真实客户 |
| ❌ 分析竞品市场份额 | ✅ 找到"我是成都的，多少钱"这样的评论 |
| ❌ 监控竞品内容策略 | ✅ 找到"我家屋顶能装吗"这样的用户 |
| ❌ 比较竞品产品优劣 | ✅ 把竞品评论区变成 PV_OS 的客户来源 |

### 1.2 在 PV_OS 链路中的位置

```
竞品账号库（本文档）
    │  被读取
    ▼
comment_collector_agent
    │  采集评论区
    ▼
comment_intent_model
    │  语义理解
    ▼
comment_analyzer → lead_scoring_agent → CRM
```

> 引用：`COMMENT_COLLECTOR_AGENT_DESIGN.md` §八 系统连接。

## 二、账号分类体系

### 2.1 四级分类总览

围绕城市小区光伏客户评论集中度，建立四级分类。

> 引用自 `COMPETITOR_SCORE_RULE.md` §二 六维评分、`COMMENT_COLLECTOR_AGENT_DESIGN.md` §三 账号发现模型。

#### 一级：全国光伏品牌

| 属性 | 说明 |
|------|------|
| 典型账号 | 正泰安能、天合富家、阳光新能源、创维光伏、晶科能源等全国性品牌官方号 |
| 内容特征 | 品牌宣传、大型项目案例、行业活动、产品发布 |
| 评论特征 | 覆盖面广，评论分散在全国各地。川渝黔客户占比被全国基数稀释 |
| 城市客户密度 | 🟡 中（需按 IP 属地过滤川渝黔） |
| 采集策略 | 仅采集 IP 属地 = 川渝黔的评论 |

#### 二级：区域安装商

| 属性 | 说明 |
|------|------|
| 典型账号 | 川渝黔本地光伏安装公司/团队（如"成都光伏老王""重庆阳光能源""贵阳鑫达光伏"类账号） |
| 内容特征 | 本地安装案例实拍、客户见证、施工过程、并网展示 |
| 评论特征 | **评论区是城市小区客户的最高密度来源**。"成都这边怎么联系""重庆江北能上门吗""绵阳多少钱" |
| 城市客户密度 | 🔴 极高 |
| 采集策略 | 全量采集，最高频率 |

#### 三级：城市小区案例账号

| 属性 | 说明 |
|------|------|
| 典型账号 | 以高端住宅光伏案例为核心内容的账号 |
| 内容特征 | **住宅案例**：别墅光伏安装前后对比、业主采访；**小区安装案例**：某小区多户安装合集；**屋顶改造案例**：阳光房改光伏、露台做光伏、花园洋房加装 |
| 评论特征 | "我家也是别墅""叠拼能做吗""阳光房改光伏要多少钱""成都有案例吗" |
| 城市客户密度 | 🔴 极高（内容本身筛选了受众） |
| 采集策略 | 全量采集，最高频率 |

#### 四级：装修/别墅/住宅改造账号

| 属性 | 说明 |
|------|------|
| 典型账号 | 高端住宅装修公司、别墅改造设计师、阳光房定制工作室 |
| 内容特征 | 别墅装修设计、阳光房改造、花园洋房翻新、露台利用方案——交叉涉及光伏/新能源话题 |
| 评论特征 | "这个阳光房能做光伏吗""装修含光伏设计吗""光伏和阳光房怎么结合" |
| 城市客户密度 | 🟡 中高（受众本身为有装修预算的城市业主） |
| 采集策略 | 仅采集含"光伏""太阳能""新能源"关键词的视频评论 |

### 2.2 分类与采集策略总览

| 层级 | 城市客户密度 | 采集优先级 | 采集方式 |
|:----:|:----------:|:--------:|---------|
| 二级（区域安装商） | 🔴 极高 | P0 | 全量评论，最高频率 |
| 三级（城市案例账号） | 🔴 极高 | P0 | 全量评论，最高频率 |
| 一级（全国品牌） | 🟡 中 | P1 | 按 IP 属地过滤川渝黔 |
| 四级（装修别墅账号） | 🟡 中高 | P1 | 仅光伏相关视频评论 |

## 三、账号价值判断模型

### 3.1 核心原则

**不要使用账号名称直接判断价值。** 账号名称可以伪装——"成都光伏专家"可能实际是外地品牌，"阳光房改造日记"可能是纯内容号无商业行为。

真正判断依据来自账号的实际内容表现：

| 评价维度 | 权重 | 如何判断 | 判断依据 |
|---------|:----:|---------|---------|
| **内容相关性** | 25% | 账号内容与光伏/家庭能源/住宅安装的相关程度 | 最近 30 条作品的主题分析 |
| **城市客户比例** | 25% | 评论区中川渝黔城市客户的占比 | 评论 IP 属地 + 评论文本区域信号 |
| **评论质量** | 20% | 评论中真实需求出现的密度（询价/咨询/要联系方式） | 评论语义分析 |
| **真实需求出现概率** | 20% | 每条视频平均有多少条评论含有购买意向 | 需求信号占比（想装/报价/多少钱/怎么联系） |
| **商业价值** | 10% | 评论区客户的高价值房屋场景占比 | 别墅/叠拼/阳光房/大平层/花园洋房提及频率 |

### 3.2 五维判断矩阵

| 维度 | 高分特征（≥80%） | 中分特征（50-79%） | 低分特征（<50%） |
|------|---------------|-----------------|---------------|
| 内容相关性 | 90%+ 内容围绕光伏安装 | 50%+ 光伏，混合装修/能源 | 偶尔提光伏，主体无关 |
| 城市客户比例 | IP 属地 70%+ 为川渝黔城市 | IP 属地 40-70% 为川渝黔 | IP 属地 <40% 为川渝黔 |
| 评论质量 | 每 10 条评论有 3+ 条询价/咨询 | 每 10 条有 1-2 条 | 几乎没有询价类评论 |
| 真实需求概率 | 15%+ 评论含购买信号 | 5-15% | <5% |
| 商业价值 | 别墅/叠拼/阳光房高频出现 | 普通住宅为主 | 无房屋场景信号 |

> 引用：`COMPETITOR_SCORE_RULE.md` §2.2 城市家庭光伏匹配、§2.5 评论区需求价值。

### 3.3 入库判断标准

| 判断结论 | 条件 | 采集动作 |
|:--------:|------|---------|
| ✅ 入库 P0 | 二级或三级分类 + 五维中 ≥3 维为高分 | 全量采集，每 6 小时 |
| ✅ 入库 P1 | 一级或四级分类 + 城市客户比例 ≥40% | 按条件采集，每日 |
| ⚪ 观察 | 五维中 ≥2 维为低分 | 不主动采集，定期复评 |
| ❌ 不入库 | 内容相关性 <30% 且评论质量低分 | 标记排除 |

## 四、账号字段设计

### 4.1 账号主表字段

对齐 `COMPETITOR_DISCOVERY_ALGORITHM.md` §9.3 的 `competitor_master.csv` 字段定义，并扩展 comment_collector 所需字段。

| # | 字段 | 类型 | 必填 | 说明 |
|:-:|------|------|:--:|------|
| 1 | `account_id` | string | ✅ | 内部唯一 ID（格式：`平台_账号ID`） |
| 2 | `platform` | enum | ✅ | douyin / xiaohongshu / bilibili / kuaishou |
| 3 | `account_name` | string | ✅ | 账号昵称 |
| 4 | `category` | enum | ✅ | national_brand / regional_installer / city_case / renovation |

| 7 | `priority` | enum | ✅ | P0 / P1 / P2 |
| 8 | `collection_status` | enum | ✅ | active / paused / pending / inaccessible |

### 4.2 字段值域定义

| 字段 | 允许值 |
|------|--------|

| `city_customer_ratio` | `high`（≥70%）、`medium`（40-70%）、`low`（<40%）、`unknown`（未评估） |
| `demand_density` | `high`（≥15% 评论含购买信号）、`medium`（5-15%）、`low`（<5%）、`unknown` |

## 五、视频采集关系

### 5.1 完整链路

```
账号（competitor_master.csv）
    │
    ▼
视频内容（按账号获取最近 N 条视频）
    │  AI 分析视频标题和内容主题
    │  判断是否属于目标视频类型
    ▼
评论数据（进入采集的视频 → 全量采集评论）
    │
    ▼
comment_collector_agent（输出 02_DATA/raw/）
```

### 5.2 核心认知

**账号不是最终目标，评论中的真实需求用户才是目标。**

```
账号 → 只是一个"容器"
视频 → 评论区才是"矿藏"
评论 → 其中真实需求的用户才是"矿"
```

一个 S 级竞品账号如果评论区没有真实客户询价，对 PV_OS 没有价值。
一个粉丝不多但评论区高密度出现"多少钱""怎么装"的账号，P0 优先级。

### 5.3 视频筛选规则

引用自 `COMMENT_COLLECTOR_AGENT_DESIGN.md` §四 视频发现模型：

| 视频类型 | 标题特征 | 采集方式 |
|---------|---------|---------|
| 光伏安装案例 | "XX安装实拍""XX并网成功" | 全量评论 |
| 屋顶改造 | "屋顶光伏改造""露台做光伏" | 全量评论 |
| 家庭能源 | "家庭光伏发电""电费从X到Y" | 全量评论 |
| 收益分析 | "回本要多久""划不划算" | 全量评论 |
| 安装价格 | "装一套多少钱""价格对比" | 全量评论 |
| 住宅装修（含光伏） | "别墅装修""阳光房改造" | 仅光伏相关评论 |

## 六、与评论采集连接

### 6.1 comment_collector_agent 如何读取账号库

```
comment_collector_agent 启动
    │
    ▼
Step 1: 读取 competitor_master.csv
    │  filter: collection_status = active
    │  filter: priority IN (P0, P1)
    │  order: priority ASC, city_customer_ratio DESC
    │
    ▼
Step 2: 按 category 选择采集策略
    ├── regional_installer / city_case → 全量评论，高频率
    ├── national_brand → IP 属地过滤川渝黔
    └── renovation → 仅光伏相关视频
    │
    ▼
Step 3: 按平台分组
    │  分配给对应 connector
    │
    ▼
Step 4: 获取视频 → 筛选 → 采集评论 → 输出
```

### 6.2 账号库维护流程

```
新增账号
    │
    ▼
五维价值判断（§三）
    │
    ▼
确定 category + priority + collection_status
    │
    ▼
写入 competitor_master.csv
    │
    ▼
comment_collector_agent 下次采集周期自动读取
```

### 6.3 账号状态流转

```
pending（待审核）
    │  五维判断通过
    ▼
active（正常采集）
    │  连续 3 次采集周期评论数 <5 → 暂停
    ▼
paused（暂停）
    │  定期复评（每 30 天）
    ├── 内容恢复 → active
    └── 持续无产出 → 降级
```

## 七、城市小区客户重点规则

### 7.1 核心理念

**不要要求用户必须说特定关键词才能被发现。**

真实城市小区光伏客户在评论区通常这样说话：

| 用户真实表达 | PV_OS 必须能捕获 | 不应要求用户说 |
|------------|:---------------:|-------------|
| "我是成都的，可以装吗" | ✅ | "成都光伏安装" |
| "我家在重庆江北" | ✅ | "重庆家庭光伏" |
| "大概多少钱" | ✅ | "光伏报价查询" |
| "可以安装吗" | ✅ | "安装光伏需求" |
| "我家屋顶可以吗" | ✅ | "屋顶光伏条件咨询" |
| "多久回本" | ✅ | "光伏投资收益分析" |
| "怎么联系你" | ✅ | "联系方式获取" |
| "能上门看看吗" | ✅ | "上门勘测需求" |

### 7.2 账号选择如何影响客户发现

| 如果只采集这类账号 | 能发现的客户类型 | 漏掉的客户 |
|-----------------|---------------|----------|
| 账号名含"光伏" | 只发现已经知道"光伏"这个词的用户 | 说"太阳能""发电""省电"的用户 |
| 只搜"别墅 光伏" | 只说"我家三层楼""大房子"的用户 | 说"大平层""花园洋房""顶楼带露台"的用户 |
| 只看纯光伏账号 | 精准但有天花板 | 装修账号下的交叉需求用户 |

### 7.3 正确的账号选择策略

```
P0 账号（区域安装商 + 城市案例）
    ↓ 采集所有评论
能发现：
    - "我是成都的，多少钱？"
    - "我家屋顶能装吗？"
    - "怎么联系？"
    - "装了真的省电吗？"
    - 任何自然语言表达的潜在需求

P1 账号（装修/别墅改造）
    ↓ 采集光伏相关视频评论
能发现：
    - "这个阳光房能做光伏吗？"
    - "装修含光伏吗？"
    - 交叉需求用户（装修预算 → 光伏需求）
```
