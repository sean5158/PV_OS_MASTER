## 一、平台来源地图

### 1.1 抖音

| 维度 | 说明 |
|------|------|
| **平台用户特点** | 全年龄段覆盖，日活最高。川渝黔本地内容生态成熟，大量本地光伏安装商在此运营 |
| **城市客户价值** | 🔴 最高。城市家庭用户基数最大，本地案例视频下的"我家能不能装"需求最直接 |
| **光伏相关内容类型** | 安装实拍（别墅/自建房/阳光房）、施工过程、发电数据展示、并网流程科普、客户案例访谈 |
| **推荐采集方向** | 川渝黔本地安装商账号的别墅/叠拼/阳光房案例视频评论区；"光伏安装""家庭光伏"关键词搜索结果 |

**典型高价值评论场景（城市小区客户）：**

```
视频：成都别墅光伏安装实拍
评论区：→ "我在武侯区别墅也想装一套，能上门看看吗？"
        → "这个多少钱？包并网吗？"
        → "成都能装吗？我这边是叠拼"
```

> 引用：`CUSTOMER_SCORE_MODEL.md` §2.3（成都核心区 20 分）、§2.4（别墅 20 分 / 叠拼 18 分）。

### 1.2 小红书

| 维度 | 说明 |
|------|------|
| **平台用户特点** | 一线/新一线城市女性为主，消费决策力强，习惯主动询问品牌/价格/推荐 |
| **城市客户价值** | 🟠 高。阳光房/露台/花园洋房内容独特生态，用户"求推荐"类评论商业意图自然且转化率高 |
| **光伏相关内容类型** | 阳光房改造日记、露台光伏装修分享、家庭能源改造攻略、别墅装修记录、光伏避坑指南 |
| **推荐采集方向** | 阳光房/露台改造类笔记评论区；"光伏"+"装修"+"别墅"+"阳光房"组合搜索 |

**典型高价值评论场景（城市小区客户）：**

```
笔记：我家花园洋房装了光伏，一年省了XXXX元
评论区：→ "求安装师傅联系方式！"
        → "我家也是花园洋房，这个要多少钱？"
        → "贵阳这边有推荐的吗？"
```

> 引用：`CUSTOMER_SCORE_MODEL.md` §2.4（阳光房 18 分 / 花园洋房 15 分）。

### 1.3 B站

| 维度 | 说明 |
|------|------|
| **平台用户特点** | 一二线城市年轻用户为主，内容深度导向，偏好技术原理和实测数据 |
| **城市客户价值** | 🟡 中高。用户以"了解技术"和"比较方案"为主，属于"明确兴趣"层级（需求强度 20 分），长期培育价值高 |
| **光伏相关内容类型** | 光伏原理科普、家庭光伏实测、储能系统测评、光伏安装避坑、电费对比分析 |
| **推荐采集方向** | 光伏科普/实测视频评论区；"家庭光伏""光伏储能""屋顶发电"关键词搜索 |

**典型高价值评论场景：**

```
视频：家庭光伏实测：一年发电XXXX度，回本要多久？
评论区：→ "成都这边日照条件和你那边比怎么样？"
        → "我也想做一套，有没有推荐的安装公司？"
        → "叠拼屋顶面积够吗？大概能装多少千瓦？"
```

> 引用：`CUSTOMER_SCORE_MODEL.md` §2.2（明确兴趣 20 分）。

### 1.4 快手

| 维度 | 说明 |
|------|------|
| **平台用户特点** | 下沉市场为主，光伏内容以农村自建房为主流，城市小区客户浓度显著低于抖音/小红书 |
| **城市客户价值** | 🟡 中低。城市客户分散，但农村经营场景（民宿、农家乐、养殖）按商业客户处理有挖掘价值 |
| **光伏相关内容类型** | 农村自建房光伏安装、养殖场光伏、农家乐光伏、小商业光伏 |
| **推荐采集方向** | 仅采集含"别墅""阳光房""大平层""民宿""农家乐""养殖"关键词的评论，不泛采集 |

**城市客户相关评论场景：**

```
视频：民宿装光伏到底划不划算？
评论区：→ "我成都这边也开了个民宿，可以装吗？"
```

> 引用：`CUSTOMER_SCORE_MODEL.md` §8.3（农村经营场景按商业客户处理）。

### 1.5 视频号

| 维度 | 说明 |

| **城市客户价值** | ⚪ 低。公开评论少，商业意图弱，且几乎不可自动化采集 |
| **光伏相关内容类型** | 公众号关联视频、光伏知识分享、企业品牌展示 |

## 四、评论价值信号

### 4.1 信号定义原则

以下信号标签仅用于**采集阶段的优先级排序**，不创建新的评分规则。最终评分由 `CUSTOMER_SCORE_MODEL.md` 五维评分模型决定。

### 4.2 高价值信号（采集阶段标记为 `signal: high`，pipeline 优先处理）

| 信号类别 | 识别特征 | 来源 | 对应评分维度 |
|---------|---------|------|:----------:|
| **询价信号** | "多少钱""报价""费用""贵不贵""预算""价格" | CUSTOMER_SCORE_MODEL §2.2（30-40 分） | 需求强度 |
| **安装咨询** | "想装""怎么装""安装流程""能装吗""帮我看看" | CUSTOMER_SCORE_MODEL §2.2（30-40 分） | 需求强度 |
| **收益咨询** | "效果怎么样""发电多少""够用吗""能省多少" | CUSTOMER_SCORE_MODEL §2.2（20 分） | 需求强度 |
| **面积咨询** | "我家 XX 平""能装多少千瓦""屋顶面积 XX" | CUSTOMER_SCORE_MODEL §2.2（30-40 分） | 需求强度 + 房屋场景 |
| **位置咨询** | "成都这边有吗""重庆能装吗""XX 区可以上门吗" | CUSTOMER_SCORE_MODEL §2.3 | 区域价值 |
| **联系方式需求** | "怎么联系""电话多少""私信我""加微信" | CUSTOMER_SCORE_MODEL §2.2（40 分） | 需求强度 |
| **上门请求** | "能上门看看吗""来我家测一下""能不能来看看" | CUSTOMER_SCORE_MODEL §2.2（40 分） | 需求强度 |
| **房屋场景明确** | "我家别墅""叠拼""大平层""花园洋房""阳光房" | CUSTOMER_SCORE_MODEL §2.4（15-20 分） | 房屋场景 |

### 4.3 普通价值信号（标记为 `signal: medium`，正常排队处理）

| 信号类别 | 识别特征 | 来源 |
|---------|---------|------|
| **了解信号** | "了解一下""看看""关注了""收藏了" | CUSTOMER_SCORE_MODEL §2.2（10 分） |
| **比较信号** | "和 XX 比怎么样""哪个好""区别是什么" | CUSTOMER_SCORE_MODEL §2.2（10-20 分） |
| **关注信号** | "光伏真的省电吗""太阳能靠谱吗""能用多久" | CUSTOMER_SCORE_MODEL §2.2（20 分） |
| **区域模糊** | IP 属地为川渝黔但未明确城市 | COMMENT_TIME_AND_MATCH_RULE §六 |

### 4.4 低价值信号（标记为 `signal: low`，不丢弃但降低 pipeline 优先级）

| 信号类别 | 识别特征 | 来源 |
|---------|---------|------|
| **无需求讨论** | 纯技术讨论、行业新闻评论、表情回复、与光伏安装无关 | CUSTOMER_SCORE_MODEL §2.2（0 分） |
| **纯情绪表达** | "好看""不错""厉害了"等无实质内容 | 采集阶段判断 |
| **免费政策期待** | "免费装""国家免费""什么时候免费" | CUSTOMER_SCORE_MODEL §2.2 负向修正 |
| **补贴咨询** | "有没有补贴""补贴多少""政府补助" | CUSTOMER_SCORE_MODEL §2.2 负向修正 |

### 4.5 信号与采集动作对照

| 信号等级 | 采集动作 | pipeline 优先级 |
|:--------:|---------|:--------------:|
| `high` | 正常采集 | 排入队列最前，优先进入 `evaluate_comment_time` |
| `medium` | 正常采集 | 正常排队 |
| `low` | 正常采集（不丢弃） | 排入队列末尾，仅保存分析 |
