# PV_OS_COMMENT_INTENT_MODEL

版本：V1.0
日期：2026-07-13
用途：PV_OS 城市小区光伏客户评论意图识别模型——通过 AI 语义理解从原始评论中识别真实购买意向，作为 comment_analyzer 的前置语义层

> 设计原则：仅基于已有固化规则文件设计，不自行创造业务规则。
> 核心聚焦：城市小区光伏客户。
> 定位：采集后 → AI 语义理解 → 识别真实购买意向 → 输出给 comment_analyzer → 进入 lead_scoring_agent。
> 不替代评分模型。评分由 CUSTOMER_SCORE_MODEL.md 五维评分负责。
>
> 依赖规则：
> - `PV_OS_MASTER_CONTEXT.md` — 项目全局上下文
> - `PV_OS_COMMENT_COLLECTION_STRATEGY.md` V2.0 — 采集策略
> - `COMMENT_COLLECTION_AUDIT.md` — 采集能力审计
> - `02_DATA/04_COMMENT_DATABASE/COMMENT_ANALYZER_RULE.md` — 评论分析规则
> - `02_DATA/06_SCORE_MODEL/CUSTOMER_SCORE_MODEL.md` — 五维评分模型
> - `03_AI_AGENT/agents/comment_analyzer/agent.yml` — 下游分析入口
> - `03_AI_AGENT/agents/lead_scoring_agent/agent.yml` V2.1 — 下游评分入口

## 一、模型定位

### 1.1 PV_OS 完整链路中的位置

```
视频平台
    │
    ▼
评论采集（02_DATA/01_COLLECTION/）
    │
    ▼
┌─────────────────────────────────────────────────┐
│  评论意图识别模型（本文档定义）                     │  ← 前置语义层
│  功能：语义理解 → 意图分类 → 客户识别 → 输出       │
│  位置：采集后，comment_analyzer 前                 │
└─────────────────────────────────────────────────┘
    │
    ▼
comment_analyzer（03_AI_AGENT/agents/comment_analyzer/）
    │
    ▼
lead_scoring_agent（03_AI_AGENT/agents/lead_scoring_agent/）
    │
    ▼
CRM（05_CUSTOMER_CRM/）
```

### 1.2 核心原则

**不是通过简单关键词寻找客户。** 关键词匹配只能命中显性表达，无法理解语义和语境。

本模型解决的核心问题：

| 关键词匹配的局限 | 语义理解的解决 |
|----------------|-------------|
| "想装光伏" 命中，"我家在成都" 不命中 | "我家在成都" + 光伏视频上下文 → 可能是咨询意向 |
| 容易误判广告和机器评论 | 语义分析自然度，识别真实人类表达 |
| 不能理解否定和讽刺 | "装了三年没效果" → 不是购买意向，是负面反馈 |

### 1.3 与 comment_analyzer 的关系

| 维度 | 意图识别模型（本文档） | comment_analyzer |
|------|---------------------|-----------------|
| 职责 | 语义理解、意图分类、客户识别、信号提取 | 分析维度补全、初步评分、标签生成 |
| 输入 | 原始评论 + 上下文（平台/IP/视频标题） | 意图模型输出 |
| 输出 | 结构化客户意图数据 | lead_scoring_agent 需要的完整字段 |

## 二、真实客户评论特征

### 2.1 用户不会怎么表达

真实用户在竞品评论区不会使用以下方式表达：

| 不会说的 | 原因 |
|---------|------|
| "我是高端小区客户" | 用户不会主动给自己贴标签 |
| "我是别墅客户" | 可能说"我家""独栋"但不自称为"别墅客户" |
| "我有消费能力" | 消费能力通过询问报价/安装意愿间接体现 |
| "我是城市小区客户" | 城市身份通过地名/语境自然流露 |
| "我的需求强度是40分" | 需求通过"想装""多少钱"等自然语言表达 |
| "我是S级客户" | 等级是 AI 评分结果，不是用户自我描述 |

### 2.2 用户通常会怎么表达

真实城市小区光伏客户在竞品评论区通常以以下自然方式表达：

| 真实表达方式 | 示例 | 背后的意图 |
|------------|------|----------|
| **表明位置** | "我是成都的""在重庆这边""贵阳能装吗" | 区域咨询，区域价值判断依据 |
| **确认可行性** | "我家可以装吗""这种屋顶能装光伏吗""顶楼能做吗" | 安装条件咨询，房屋场景判断依据 |
| **询价** | "大概多少钱""装一套要多少""预算多少够" | 价格敏感但具备购买意向 |
| **问收益** | "收益怎么样""多久回本""能省多少电费" | 投资决策阶段，明确兴趣 |
| **找案例** | "有没有成都的安装案例""能看看你家装的吗" | 购买决策前的研究行为 |
| **问时间** | "什么时候可以安装""安装要多久" | 已进入执行阶段，极强需求 |
| **要联系方式** | "怎么联系""给个电话""私信你了" | 购买意向最明确的信号 |
| **描述房屋** | "我家两层楼""有个露台""花园洋房带屋顶" | 房屋场景的间接描述 |
| **对比选择** | "选哪个牌子好""和XX安装商比怎么样" | 处于比较决策阶段 |

### 2.3 AI 语义理解要点

真实客户评论的语义特征（与广告/机器人/无关评论区分）：

| 维度 | 真实客户 | 广告/机器人 |
|------|---------|:----------:|
| 语言自然度 | 口语化、有语气词、有停顿、可能用方言 | 模板化、机械化、无个性 |
| 问题方向 | 以"我"为中心（"我能""我家""我这边"） | 以推广为中心（"加微信""联系我""点击"） |
| 信息完整度 | 经常不完整，需要上下文推断 | 高度结构化，信息完整 |
| 互动感 | 像在和人对话（"能告诉我吗""谢谢"） | 单向推送 |
| 情绪信号 | 有期待、焦虑、犹豫等真实情绪 | 无情绪或过度亢奋 |

> 引用：`CUSTOMER_SCORE_MODEL.md` §2.5 用户真实性维度（0-10 分）。

## 三、客户意图分类

### 3.1 分类原则

以下分类仅用于意图识别阶段的语义归类，**不替代 `CUSTOMER_SCORE_MODEL.md` 的评分**。评分由 lead_scoring_agent 的五维模型负责。

> 引用自 `CUSTOMER_SCORE_MODEL.md` §2.2 需求强度等级（40/30/20/10/0 分）和 `comment_analyzer/agent.yml` intent_level 定义（0-3）。

### 3.2 高意向（intent: high）

用户已表现出明确的购买或深度咨询行为。comment_analyzer 对应 `intent_level: 2-3`。

| 意图子类 | 语义特征 | 示例 | 关键词参考（CUSTOMER_SCORE_MODEL §2.2） |
|---------|---------|------|---------------------------------------|
| **咨询安装** | 明确表达"想装""要装""准备装"，询问安装流程或条件 | "我想在成都家里装一套光伏" | 想装类（40 分） |
| **询价** | 主动询问价格、费用、预算 | "装一套大概多少钱？" | 报价类 + 询价类（30-40 分） |
| **区域安装** | 确认自己所在城市能否安装 | "绵阳这边能装吗？""重庆能上门吗？" | 上门类（40 分）+ 区域匹配 |
| **面积/型号咨询** | 询问自家面积能装多少、什么规格 | "我家100平屋顶能装多大？" | 发电量类（20-40 分） |
| **收益咨询** | 关注发电量、回本周期、省电金额 | "一年能发多少度电？多久回本？" | 发电量类（20 分） |
| **要联系方式** | 直接要求联系、私信、加微信 | "给个联系方式""私你了" | 极强需求（40 分） |

### 3.3 中意向（intent: medium）

用户有光伏兴趣但未进入购买决策阶段。comment_analyzer 对应 `intent_level: 1-2`。

| 意图子类 | 语义特征 | 示例 | 关键词参考 |
|---------|---------|------|----------|
| **关注收益** | 看视频后对省电/收益产生兴趣 | "真的能省这么多吗？" | 效果类（20 分） |
| **比较方案** | 在多个方案间比较，未下决定 | "和 XX 品牌比怎么样？" | 普通了解（10-20 分） |
| **了解政策** | 询问补贴政策、并网政策（非免费期待） | "现在装还有补贴吗？" | 补贴咨询（需区分免费期待） |
| **观望型** | "了解一下""看看""关注了" | "先了解一下，以后再考虑" | 观望型（负向修正 -10） |

### 3.4 低意向（intent: low）

用户评论与购买意向无关或仅有极弱关联。comment_analyzer 对应 `intent_level: 0-1`。

| 意图子类 | 语义特征 | 示例 |
|---------|---------|------|
| **围观** | "看看""不错""厉害" | "这个做得好""厉害了" |
| **质疑** | 对光伏效果或商业模式的质疑 | "光伏就是骗人的""没用" |
| **无明确需求** | 评论与光伏安装完全无关 | "这视频拍得真好" |
| **纯情绪** | 表情、点赞、无实质文本 | "👍👍👍" |

## 四、城市客户识别

### 4.1 城市客户不是关键词定义

城市客户不能通过简单的"城市""小区"关键词来识别。城市客户身份的判断是**多信号综合推断**的结果。

### 4.2 四维综合判断

> 引用自 `PV_OS_MASTER_CONTEXT.md` §三 目标客户模型（城市客户通过房屋 + 区域乘积效应自然筛选）、`CUSTOMER_SCORE_MODEL.md` §2.3 区域价值 + §2.4 房屋场景。

| 判断维度 | 如何从评论中识别 | 示例 |
|---------|----------------|------|
| **地区表达** | 用户提到川渝黔城市/区县名，或 IP 属地在核心市场 | "成都""重庆江北""贵阳观山湖区"；IP：四川 |
| **房屋表达** | 用户描述住宅类型，暗示城市生活场景 | "我家叠拼""花园洋房""大平层""小区顶楼" |
| **安装条件** | 用户确认"能装吗""屋顶条件"，暗示有独立屋顶的城市住宅 | "我们小区让装吗""顶楼可以做吗" |
| **购买行为** | 询价、问流程、要上门，暗示有支付意愿和决策权 | "大概多少钱""能上门看看吗" |

### 4.3 城市客户信号强度

| 信号组合 | 城市客户概率 | 说明 |
|---------|:----------:|------|
| 地区明确（川渝黔城市） + 房屋高价值（别墅/叠拼/阳光房/大平层） | 🔴 极高 | 典型的城市小区客户 |
| 地区明确（川渝黔城市） + 房屋普通但主动询价 | 🟠 高 | 城市普通住宅客户，仍有购买意向 |
| 地区模糊（仅省级） + 房屋高价值 | 🟡 中 | 有城市客户特征但需进一步确认 |
| 地区明确（川渝黔城市） + 无房屋信号 + 无购买行为 | 🟢 低 | 城市用户但无光伏需求 |
| IP 属地 = 川渝黔 + 无任何城市/房屋信号 | ⚪ 极低 | 仅 IP 推测，置信度不足 |

## 五、房屋场景识别

### 5.1 识别原则

房屋场景通过评论语义推断，不是简单关键词匹配。用户可能用各种自然语言描述同一类住宅。

> 引用自 `COMMENT_ANALYZER_RULE.md` §五 房屋场景识别（高/中价值二分法）和 `CUSTOMER_SCORE_MODEL.md` §2.4 房屋场景精确分值。

### 5.2 别墅 / 独栋

| 用户可能的表达方式 | 语义理解要点 |
|-----------------|------------|
| "我家是独栋""三层楼那种""自己建的房子"（城市区域语境下） | "独栋""三层" + 城市区域 → 别墅 |
| "在XX区别墅区" | 直接命中 |
| "我家带院子""车库可以吗" | 独立住宅信号 |

> 分值：20（CUSTOMER_SCORE_MODEL §2.4）

### 5.3 改善型住宅（叠拼/阳光房/大平层/花园洋房）

| 用户可能的表达方式 | 语义理解要点 |
|-----------------|------------|
| "我家叠拼，上下两层" | 叠拼 |
| "有个阳光房/玻璃房想利用" | 阳光房 |
| "大平层，客厅很大那种" | 大平层 |
| "花园洋房一楼带花园" | 花园洋房 |
| "顶楼带露台""楼顶属于我家的" | 露台（18 分） |

> 分值：叠拼/阳光房/露台 18，大平层/花园洋房 15（CUSTOMER_SCORE_MODEL §2.4）

### 5.4 普通城市住宅

| 用户可能的表达方式 | 语义理解要点 |
|-----------------|------------|
| "普通小区""楼顶可以装吗""我们小区有没有装的" | 普通住宅顶楼 → 10 分 |
| "家里想装""我家能用吗"（无特殊房屋描述 + 城市语境） | 普通住宅 → 5 分 |

> 分值：普通住宅顶楼 10，普通住宅 5（CUSTOMER_SCORE_MODEL §2.4）

### 5.5 其他场景

| 类型 | 语义特征 | 分值 | 来源 |
|------|---------|:--:|------|
| 农村自建房 | "自建房""农村老家""宅基地" | 12 | CUSTOMER_SCORE_MODEL §2.4 |
| 商业建筑 | "民宿""客栈""茶楼""棋牌室""门店" | 按商业客户 | CUSTOMER_SCORE_MODEL §8.3 |
| 未知 | 无任何房屋信号 | — | 进入 scoring 阶段由 customer_type 辅助判断 |

## 六、消费能力判断

### 6.1 禁止收入判断

PV_OS 不通过评论推断用户收入水平、职业、资产。原因：

- 评论中缺乏可靠收入信号
- 收入推断误差大，易造成偏见
- 违反 `comment_schema.md` 隐私原则（"只保存公开信息"）

### 6.2 消费能力间接判断依据

消费能力通过以下方式间接体现，均来自已有规则：

> 引用自 `PV_OS_MASTER_CONTEXT.md` §3.4 消费能力判断。

| 依据 | 判断逻辑 | 来源 |
|------|---------|------|
| **房屋条件** | 别墅(20 分) → 普通住宅(5 分)，房屋价值反映资产水平 | CUSTOMER_SCORE_MODEL §2.4 |
| **区域价值** | 成都核心区(20 分) → 次级城市(13 分)，城市能级反映消费力 | CUSTOMER_SCORE_MODEL §2.3 |
| **安装意愿** | 主动询价(40 分) = 具备支付意愿；免费期待(-25 分) = 无支付意愿 | CUSTOMER_SCORE_MODEL §2.2 |
| **需求强度** | 极强需求(40 分) > 明确兴趣(20 分) > 普通了解(10 分) | CUSTOMER_SCORE_MODEL §2.2 |
| **真实性** | 真实用户(10 分) > 可疑用户(0 分)，消费判断只在真人基础上 | CUSTOMER_SCORE_MODEL §2.5 |

### 6.3 农村客户消费能力特别说明

引用自 `CUSTOMER_SCORE_MODEL.md` §8.3：

> 农村经营场景（民宿、农家乐、养殖、合作社）被识别为商业支付能力，按商业客户处理。

农村客户消费能力不因"农村"标签自动降低。判断标准：

> 购买意愿 > 房屋条件 > 区域 > 农村标签

## 七、评论过滤规则

### 7.1 过滤原则

采集阶段不丢弃评论（除明显无效数据）。意图识别阶段识别出无价值评论类型后，标记但不丢弃，保留供数据分析。

> 引用自 `COMMENT_DATA_LIFECYCLE_RULE.md`：所有评论进入长期资产库。

### 7.2 广告评论识别

| 特征 | 语义信号 |
|------|---------|
| 推广意图 | "加微信""联系我""点击主页""找我安装" |
| 品牌植入 | "XX光伏公司""XX品牌最好" |
| 联系方式 | 直接留手机号/微信号 |
| 语气模板化 | 高度结构化，缺乏个人化表达 |
| 跨视频重复 | 同一内容出现在多个视频评论区 |

标记：`is_advertisement: true`，进入分析库但不进入客户线索库。

### 7.3 机器人/水军评论识别

| 特征 | 语义信号 |
|------|---------|
| 内容模板化 | "光伏真好""太厉害了""值得推荐"等通用赞美 |
| 与视频无关 | 内容与光伏/家庭能源/装修无关 |
| 高频重复 | 同一账号在短时间内大量相似评论 |
| 无个人语境 | 无法从中提取任何用户特征（地区/房屋/需求） |

标记：`is_bot: true`，进入分析库但不进入客户线索库。

### 7.4 同行评论识别

| 特征 | 语义信号 |
|------|---------|
| 行业术语 | 大量使用光伏行业专业术语 |
| 供应端视角 | "这个安装用什么型号组件""逆变器什么品牌" |
| 比价意图 | "我这边 XX 价格更低"（同行比价，不是客户比价） |
| 身份暗示 | "我也是做光伏的""我们公司" |

标记：`customer_type: 同行业用户`，进入 comment_analyzer 但标记低优先级。

### 7.5 无需求评论识别

| 特征 | 语义信号 |
|------|---------|
| 纯围观 | "看看""路过""不错" |
| 纯情绪 | "哈哈哈""👍" |
| 话题偏离 | 讨论与光伏完全无关的话题 |
| 负面反馈 | "装了没用""光伏骗人""后悔装了" |

标记：`intent: low`，进入分析库但不进入销售池。

## 八、输出给 comment_analyzer 的字段

### 8.1 数据结构

意图识别模型的输出作为 comment_analyzer 的结构化输入。字段定义对齐 `comment_analyzer/agent.yml` 的 `analysis` 维度。

```json
{
  "comment_text": "string        // 原始评论文本",
  "platform": "string           // douyin / xiaohongshu / kuaishou / wechat_video",
  "video_source": {
    "video_title": "string      // 评论所在视频标题（上下文）",
    "video_url": "string        // 视频链接",
    "competitor_id": "string    // 竞品账号 ID"
  },
  "region": {
    "province": "string         // 省份（四川/重庆/贵州/未知）",
    "city": "string             // 城市",
    "district": "string         // 区县（可选）",
```

## 九、与评分模型连接

### 9.1 职责边界

| 模型 | 职责 | 不负责 |
|------|------|--------|
| **意图识别模型（本文档）** | 语义理解、意图分类、客户识别、信号提取 | 评分、等级判定、CRM 路由 |
| **comment_analyzer** | 补全分析维度、初步评分(0-100)、标签生成 | 五维细化评分 |
| **lead_scoring_agent** | 五维评分（40+20+20+10+10）、S/A/B/C 等级判定、CRM 路由 | 语义理解、意图判断 |

### 9.2 数据流转

```
意图识别模型输出
    │
    ▼
comment_analyzer
    ├── 消费 intent_level → 验证并补全
    ├── 消费 customer_type → 验证并细化
    ├── 消费 province/city → 建立区域置信度
    ├── 消费 housing_type → 补全房屋场景
    └── 输出 → 05_CUSTOMER_LEADS
                │
                ▼
        lead_scoring_agent
            ├── demand_score (0-40) ← intent_level + demand_signals
            ├── region_score (0-20) ← province + city
            ├── housing_score (0-20) ← housing_type
            ├── authenticity_score (0-10) ← is_real_person + language_naturalness
            ├── time_score (0-10) ← comment_time
            └── lead_grade (S/A/B/C) ← lead_score
```

## 十、测试案例

### 10.1 测试原则

- 优先使用城市小区光伏客户场景
- 不使用农村作为主要测试
- 测试覆盖高/中/低意向分类准确性
- 测试覆盖客户识别准确率

> 引用自 `CUSTOMER_SCORE_MODEL.md` 评分标准。
