
# COMPETITOR_DISCOVERY_ALGORITHM —— 竞品账号自动发现算法 V1.0

> PV_OS 竞品自动发现系统的完整算法规则：从关键词到账号，从初筛到入库。
>
> 依赖规则：`KEYWORD_STRATEGY.md`（关键词策略）、`COMPETITOR_SCORE_RULE.md`（评分规则）



## V1.1 业务调整说明

PV_OS 竞品发现采用：

全国发现 + 区域客户筛选 模型。


说明：

竞品账号不限制地域。

原因：

优秀光伏内容账号、安装公司、个人博主可能分布全国。


但是：

PV_OS 的销售目标客户区域：

一级：
四川

二级：
重庆

三级：
贵州


因此：

账号发现范围：

全国。


客户价值判断：

通过评论区用户地域信息判断。



流程：

全国竞品账号

↓

采集视频内容

↓

采集评论区

↓

AI分析评论用户

↓

提取：

- 四川
- 重庆
- 贵州

用户

↓

进入潜客数据库。
>
> 最后更新：2026-07-11

## 一、目标定义

## 竞品账号与客户地域分离原则


竞品账号地域 ≠ 客户地域。


例如：


北京光伏博主视频：

评论：

"成都可以安装吗？"

判断：

账号：

全国竞品账号


评论用户：

四川潜客。


进入：

四川潜客池。



因此：

不能因为竞品账号不在四川，
而降低价值。

### 1.1 有效竞品账号

同时满足以下条件的账号，视为有效竞品：

| 条件 | 判定标准 |
|------|---------|
| **行业相关** | 账号内容以光伏/太阳能/新能源安装服务为主题，非纯科普、非纯产品测评 |
| **面向 C 端** | 目标客群为家庭用户或小商业主，非纯 B2B 供应链账号 |
| **具备转化行为** | 存在引导咨询、报价、联系方式等转化动作，或评论区有真实购买意向 |
| **近期活跃** | 最近 90 天内有内容更新 |
| **可观测** | 账号公开可见，内容可采集 |

### 1.2 排除范围

以下账号不纳入竞品库，即使关键词搜索命中：

| 排除类型 | 典型特征 | 处理方式 |
|----------|---------|---------|
| 纯资讯号 | 仅转发行业新闻，无原创内容 | 标记为 `info_account`，不入库 |
| 光伏组件厂商号 | 以产品批发/招商/代理为主 | 标记为 `supplier`，入行业参考库 |
| 纯科普号 | 仅做光伏原理科普，无商业行为 | 标记为 `education`，不入库 |
| 异地无关号 | 内容与光伏无关，仅昵称含关键词 | 标记为 `noise`，丢弃 |
| 已注销/私密号 | 无法获取公开内容 | 标记为 `inaccessible`，丢弃 |
| 大型能源集团官号 | 央企/上市公司官方账号，非直接竞品 | 标记为 `enterprise`，入行业参考库 |
| 个人生活号 | 偶发一条光伏内容 | 标记为 `casual`，不入库 |

### 1.3 边界判定规则

```
账号命中光伏关键词 ──→ 检查内容主线
                        │
              ┌─────────┼─────────┐
              ▼         ▼         ▼
          光伏主线    混合主线    无关主线
              │         │           │
              ▼         ▼           ▼
           进入评分   检查光伏    标记 noise
                      内容占比     丢弃
                         │
              ┌─────────┤
              ▼         ▼
           ≥30%      <30%
              │         │
              ▼         ▼
           进入评分   标记 casual
                       不进入评分
```

## 二、关键词驱动发现流程

### 2.1 人工词根输入

**Step 1：加载词根库**

从 `02_DATA/01_KEYWORD_LIBRARY/seed_keywords.yml` 加载当前有效词根（`status: active`）。

**Step 2：词根分级投放**

| 词根等级 | 投放频率 | 每词根搜索量 |
|:--------:|:--------:|:-----------:|
| S 级（≥80） | 每日 1 次 | 前 50 条结果 |
| A 级（60-79） | 每 3 日 1 次 | 前 30 条结果 |
| B 级（40-59） | 每周 1 次 | 前 20 条结果 |
| C 级（<40） | 不主动投放 | — |

**Step 3：区域词根叠加**

S/A 级词根自动与区域词组合，生成本地化搜索词：

```
词根："别墅光伏"
  ├── 区域组合 → "别墅光伏 成都"
  ├── 区域组合 → "别墅光伏 重庆"
  └── 区域组合 → "别墅光伏 贵阳"
```

### 2.2 平台搜索联想词扩展

**Step 1：获取联想词**

对每个 S/A 级词根，调用各平台搜索联想接口，获取联想词列表。

**Step 2：联想词入库**

```
联想词 → 去重（与本库已有词对比）
       → 新词 → 标记 source: "platform_suggest"
              → 记录来源平台和获取时间
              → 进入评分（KEYWORD_STRATEGY.md 第八章）
              → ≥A 级 → 投入搜索
```

**Step 3：联想词搜索**

新入库的 ≥A 级联想词在下次发现周期中作为搜索词使用，形成自扩展循环。

### 2.3 标签扩展

**Step 1：采集标签源**

从以下来源采集标签：
- 已入库 S/A 级竞品的最新作品标签
- 搜索结果中高互动作品的标签
- 平台热门话题关联标签

**Step 2：标签 → 关键词转化**

```
标签采集 → 去噪（去除非光伏标签）
         → 频次统计（按出现次数排序）
         → 频次 ≥3 → 转化为候选关键词
         → 频次 ≥5 → 标记为高价值，优先评分
         → 通过评分 → 投入搜索
```

### 2.4 视频标题关键词提取

**Step 1：标题采集**

采集已入库 S/A 级竞品最近 90 天所有作品标题。

**Step 2：分词与提取**

```
标题池 → NLP 分词 + 去停用词
       → 高频实词提取（出现 ≥3 次）
       → 与现有关键词库交叉比对
       → 未覆盖的高频词 → 候选关键词
```

**Step 3：候选关键词处理**

```
候选关键词 → 人工/ AI 判断业务相关性
           → 通过 → 进入评分 → 投入搜索
           → 不通过 → 标记为无效，不重复提取
```

### 2.5 评论区用户语言反向扩展

**Step 1：评论采集**

采集 S/A 级竞品最近 30 天评论区的用户评论。

**Step 2：需求信号提取**

参照 `KEYWORD_STRATEGY.md` 第六章规则，从评论中提取自然语言关键词。

**Step 3：反向验证**

```
评论区提取的关键词
  → 在对应平台以该词搜索
    → 搜索结果中出现新账号
      → 该词确认为有效发现词
      → 正式入库 + 评分 + 投入持续搜索
```

## 四、候选账号池设计

### 4.1 候选账号数据结构

每个通过搜索发现的账号，录入候选池时需包含以下信息：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `account_id` | string | 平台唯一账号 ID | `MS4wLjABAAAA...` |
| `platform` | enum | 平台 | `douyin` / `kuaishou` / `shipinhao` / `xiaohongshu` |
| `nickname` | string | 账号昵称 | `成都光伏老王` |
| `bio` | string | 账号简介 | `专注成都别墅光伏安装...` |
| `avatar_url` | string | 头像 URL | — |
| `follower_count` | int | 粉丝数 | `12500` |
| `content_count` | int | 作品/笔记数 | `86` |
| `source_keyword` | string | 通过哪个关键词发现 | `别墅光伏 成都` |
| `source_type` | enum | 发现方式 | `search` / `suggest` / `tag` / `comment_mention` |
| `discovery_date` | date | 发现日期 | `2026-07-11` |
| `ip_location` | string | IP 属地 | `四川` |
| `latest_interaction_avg` | int | 最近 10 条作品平均互动量 | `230` |

### 4.2 候选池去重

```
新发现账号
  ├── 与候选池已有记录比对（按 account_id）
  │   ├── 已存在 → 更新发现日期 + 追加 source_keyword
  │   └── 不存在 → 新增
  │
  └── 与竞品主库比对（按 account_id）
      ├── 已入库 → 不加入候选池，更新发现记录
      └── 未入库 → 新增候选
```

## 五、账号初筛规则

候选账号在进入正式评分前，先经过初筛，快速排除明显不符合的账号。

### 5.1 初筛漏斗

```
候选池
  │
  ├── 过滤 1：粉丝数 < 100 且作品数 < 5 → 标记 low_volume，暂不入库
  │
  ├── 过滤 2：简介不含任何光伏相关关键词 → 标记 weak_relevance，降优先级
  │
  ├── 过滤 3：IP 属地不在川渝黔且简介无区域相关信息 → 标记 out_of_region
  │
  ├── 过滤 4：最近 90 天无更新 → 标记 inactive，不入库
  │
  └── 通过初筛 → 进入 COMPETITOR_SCORE_RULE.md 评分流程
```

### 5.2 初筛与评分的衔接

```
初筛通过数量预估：
  候选池总量 → 初筛后保留约 40%-60%
              → 进入六维评分（COMPETITOR_SCORE_RULE.md）
              → 评分结果：S / A / B / C
```

### 5.3 评分规则引用

初筛通过的账号，严格按 `COMPETITOR_SCORE_RULE.md` 执行：

| 步骤 | 对应章节 | 说明 |
|------|---------|------|
| 业务匹配评分 | 2.1 | 基础门槛，<10 分直接否决 |
| 家庭光伏匹配 | 2.2 | 排除工商业竞品 |
| 高端场景权重 | 2.3 | 别墅/阳光房/小商业加分 |
| 区域权重 | 2.4 | 川渝黔加分 |
| 评论区价值 | 2.5 | 从评论区反向验证 |
| 活跃度评分 | 2.6 | 最近 7 天活跃度 |
| 等级划分 | 第三章 | S/A/B/C 四级 |
| 入库操作 | 第三章 | S/A/B 入库，C 入观察名单 |

## 六、视频级分析规则

### 6.1 分析时机

以下情况触发视频级深度分析：

| 触发条件 | 分析范围 |
|----------|---------|
| 账号初评为 A 级及以上 | 最近 30 条作品 |
| 账号初评为 B 级 | 最近 10 条作品 |
| 账号最近 7 天有新发布 | 新发布作品全部 |
| 人工指定分析 | 按指定范围 |

### 6.2 视频评分维度

对每条视频/笔记进行快速评分，判断监控优先级：

| 维度 | 满分 | 说明 |
|------|:----:|------|
| 互动率 | 30 | （点赞+评论+分享）/ 粉丝数，评估内容质量 |
| 客户意图密度 | 30 | 评论区中出现需求信号的比例 |
| 场景匹配 | 20 | 是否涉及别墅/阳光房/小商业等高价值场景 |
| 标题吸引力 | 10 | 是否含钩子词、数字、情绪表达 |
| 可复制性 | 10 | PV_OS 能否借鉴该内容结构 |
| **单视频总分** | **100** | — |

### 6.3 重点监控视频标记

| 标记 | 条件 |
|:----:|------|
| 🔥 **爆款** | 互动率 >10% 或 互动量 > 粉丝数×5% + 互动 >1000 |
| 💰 **高转化** | 评论区需求意图密度 >20%（评论含"多少钱""怎么装"等） |
| 🎯 **场景匹配** | 视频内容涉及别墅/阳光房/小商业 |
| 🆕 **新增** | 最近 7 天发布 |

> 同时满足 ≥2 个标记的视频，进入**每日监控清单**。

### 6.4 最近 7 天优先原则

- 最近 7 天发布的视频：**全面采集 + 评分**
- 8-30 天发布的视频：**抽样 50% 采集 + 评分**
- 31-90 天发布的视频：**仅记录元数据，不深入分析**
- 90 天以上的视频：**不纳入分析范围**

## 七、长期监控账号分级

### 7.1 S 级账号（核心竞品）

| 维度 | 管理策略 |
|------|---------|
| 监控频率 | 每日 |
| 内容采集 | 全部新发布视频 + 评论 |
| 分析深度 | 视频级评分 + 爆款拆解 + 策略周报 |
| 数据更新 | 每日更新账号基础数据 |
| 重评周期 | 每 15 天重新评分 |
| 降级阈值 | 连续 30 天无更新 或 重评总分 <65 |

### 7.2 A 级账号（重点关注）

| 维度 | 管理策略 |
|------|---------|
| 监控频率 | 每 3 天 |
| 内容采集 | 新发布视频的互动量 Top 50% |
| 分析深度 | 视频级评分 + 月度策略分析 |
| 数据更新 | 每 3 天更新账号基础数据 |
| 重评周期 | 每 30 天重新评分 |
| 升级阈值 | 重评总分 ≥80 → 升级 S |
| 降级阈值 | 连续 60 天无更新 或 重评总分 <45 |

### 7.3 B 级账号（常规跟踪）

| 维度 | 管理策略 |
|------|---------|
| 监控频率 | 每周 |
| 内容采集 | 新发布视频的互动量 Top 30% |
| 分析深度 | 月度汇总分析 |
| 数据更新 | 每周更新账号基础数据 |
| 重评周期 | 每 60 天重新评分 |
| 升级阈值 | 重评总分 ≥65 → 升级 A |
| 降级阈值 | 连续 90 天无更新 → 移至观察名单 |

### 7.4 C 级账号（观察名单）

| 维度 | 管理策略 |
|------|---------|
| 监控频率 | 每月 |
| 内容采集 | 仅采集基础数据变化 |
| 分析深度 | 仅月度存在感检查 |
| 数据更新 | 每月检查 1 次 |
| 重评周期 | 每 90 天检查是否升级 |
| 清理规则 | 连续 180 天无变化 → 从观察名单移除 |

```
│      ├── 去重（与候选池、主库比对）                                   │

├────────────────────────────────────────────────────────────────────┤
│ Phase 3：初筛                                                        │
│                                                                     │
│  候选池                                                              │
│      │                                                              │
│      ├── 过滤：粉丝数 <100 & 作品数 <5                                │
│      ├── 过滤：简介无光伏关键词                                       │
│      ├── 过滤：IP 属地非川渝黔                                        │
│      └── 过滤：90天无更新                                            │
│           │                                                         │
│           ▼                                                         │
│  初筛通过池（候选池的 40%-60%）                                       │
├────────────────────────────────────────────────────────────────────┤
│ Phase 4：评分                                                        │
│                                                                     │
│  初筛通过池                                                          │
│      │                                                              │
│      ▼                                                              │
│  COMPETITOR_SCORE_RULE.md 六维评分                                   │
│      │                                                              │
│      ├── 业务匹配 <10 → 直接否决，标记 C                              │
│      └── 业务匹配 ≥10 → 进入全部六维评分                             │
│           │                                                         │
│           ▼                                                         │
│  评分结果：S(≥80) / A(65-79) / B(45-64) / C(<45)                    │
├────────────────────────────────────────────────────────────────────┤
│ Phase 5：视频级分析（S/A 级 + 部分 B 级）                             │
│                                                                     │
│  采集作品列表                                                        │
│      │                                                              │
│      ├── 最近 7 天视频 → 全面采集 + 评分                             │
│      ├── 8-30 天视频 → 抽样 50%                                      │
│      └── 31-90 天视频 → 仅元数据                                     │
│           │                                                         │
│           ▼                                                         │
│  视频评分 + 重点监控标记                                              │
├────────────────────────────────────────────────────────────────────┤
│ Phase 6：入库                                                        │
│                                                                     │
│  S/A/B 级 → 02_COMPETITOR_DATABASE/ 主库                             │
│  C 级     → 观察名单（不入主库）                                      │
│                                                                     │
│  输出：评分明细 + 等级 + 监控策略                                     │
└────────────────────────────────────────────────────────────────────┘
```

## 十、后续自动化开发接口预留

### 10.1 模块化接口定义

以下接口为未来自动化脚本开发预留，当前阶段由 AI Agent 模拟执行。

```
# 关键词管理模块
get_active_keywords(grade: str) -> List[Keyword]
expand_keywords(seed: Keyword, platform: str) -> List[Keyword]
score_keyword(keyword: Keyword) -> ScoreResult

# 平台搜索模块
search_accounts(keyword: str, platform: str, depth: int) -> List[AccountBasic]
search_videos(keyword: str, platform: str, depth: int) -> List[VideoBasic]
get_account_detail(account_id: str, platform: str) -> AccountDetail

# 账号管理模块
deduplicate(candidates: List[AccountBasic]) -> List[AccountBasic]
prescreen(account: AccountDetail) -> PrescreenResult
score_account(account: AccountDetail) -> ScoreResult
classify_account(score: int) -> str  # S/A/B/C

# 视频分析模块
fetch_recent_videos(account_id: str, days: int) -> List[VideoDetail]
score_video(video: VideoDetail) -> VideoScore
mark_key_videos(videos: List[VideoScore]) -> List[VideoMark]

# 入库模块
insert_competitor(account: AccountDetail, score: ScoreResult, grade: str) -> bool
update_competitor(competitor_id: str, updates: dict) -> bool
generate_daily_report(date: str) -> Report
```

### 10.2 调用时序

```
1. get_active_keywords("S") + get_active_keywords("A")
2. 对每个 keyword:
     search_accounts(keyword, platform, depth)  × 4 平台
3. deduplicate(all_candidates)
4. 对每个 candidate:
     get_account_detail() → prescreen() → score_account() → classify_account()
5. 对 S/A 级:
     fetch_recent_videos(7) → score_video() → mark_key_videos()
6. 对 S/A/B 级:
     insert_competitor() 或 update_competitor()
7. generate_daily_report()
```
