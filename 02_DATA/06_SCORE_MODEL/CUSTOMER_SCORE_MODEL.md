> 输出对接：`05_CUSTOMER_CRM/`

## 一、模型目标

### 1.1 核心定义

**如何判断一个评论用户是否属于高价值光伏潜在客户？**

PV_OS 每天从竞品评论区发现大量用户评论。本模型通过五维量化评分，从评论用户中筛选出最可能成交的高价值光伏潜在客户，按分数排序后分配销售资源。

### 1.2 判断标准

一个评论用户被判定为"高价值光伏潜在客户"，需要同时满足：

| 条件 | 说明 |
|------|------|
| **有真实光伏安装需求** | 不是随便问问，有明确意向 |
| **在核心市场区域内** | 川渝黔地区，尤其成都 |
| **房屋场景匹配** | 别墅、阳光房、叠拼等高客单价场景优先 |
| **是真实个人用户** | 不是机器人、水军、同行试探 |
| **近期活跃** | 最近有评论活动，触达有回应可能 |

## 二、客户评分体系（100 分）

### 2.1 五维评分总览

| 维度 | 满分 | 核心问题 |
|------|:----:|---------|
| 安装需求强度 | 40 | 用户有多想装光伏？ |
| 区域匹配价值 | 20 | 用户在不在川渝黔核心市场？ |
| 房屋场景价值 | 20 | 用户住别墅还是普通住宅？ |
| 用户真实性 | 10 | 这是真人还是机器人？ |
| 时间价值 | 10 | 这个评论是最近的吗？ |
| **总分** | **100** | — |

---

### 2.2 安装需求强度（0-40 分）

| 分数 | 需求等级 | 判断标准 |
|:----:|:--------:|---------|
| **40** | 🔴 极强需求 | 明确说想安装 / 要求报价 / 要求上门测量 / 询问施工时间 |
| **30** | 🟠 强需求 | 询问多少钱 / 询问安装流程 |
| **20** | 🟡 明确兴趣 | 询问效果 / 询问发电量 |
| **10** | 🟢 普通了解 | 一般性了解光伏，无明确购买信号 |
| **0** | ⚪ 无需求 | 评论与光伏安装无关 |

#### 40 分关键字库

```
想安装类：想装、要装、准备装、打算装、考虑装、做光伏、搞一套、装一套、安一套
报价类：报价、价格、多少钱、费用、怎么收费、贵不贵、预算、成本
上门类：上门、测量、勘测、来看看、看一下、能不能来、来我家
施工类：施工时间、工期、多久装好、几天能装、什么时候装、马上装
```

#### 30 分关键字库

```
询价类：多少钱、价格、收费、费用
流程类：安装流程、怎么装、步骤、流程、手续、并网流程
```

#### 20 分关键字库

```
效果类：效果怎么样、好用吗、靠谱吗、真的省电吗、有没有用
发电量：发电多少、一天几度、够用吗、能带多少电器、多少千瓦
```

#### 需求叠加规则

当评论同时命中多个分数段的关键词时，**取最高分**。

```
例1："我想装光伏，大概多少钱？" → 命中"想装"(40分) + "多少钱"(30分) → 取 40 分
例2："光伏发电多少？安装要多久？" → 命中"发电量"(20分) + "安装流程"(30分) → 取 30 分
例3："别墅想装，能上门看看吗？" → 命中"想装"(40分) + "上门"(40分) → 取 40 分
```

---



#### 负向需求修正规则

以下信号表示用户主要关注免费政策、补贴或政府项目，
不代表明确购买意愿，需要降低销售优先级。

| 类型 | 关键词 | 修正 |
|------|--------|------|
| 免费安装期待 | 免费装、免费安装、不花钱装、国家免费装 | -25 |
| 政策等待 | 政府什么时候装、国家项目、政策下来了吗 | -20 |
| 补贴咨询 | 有没有补贴、补贴多少钱、政府补助 | -15 |
| 观望型 | 了解一下、看看政策、以后再说 | -10 |

规则：

1. 如果只有免费/政策信号，没有购买行为：
   安装需求强度最高不超过10分。

2. 如果同时出现购买信号和免费信号：
   保留购买意向，但降低优先级。

示例：

"多少钱安装？有没有补贴？"

判定：
- 存在购买意向
- 存在补贴关注
- 降低评分

"农村免费安装光伏什么时候开始？"

判定：
- 政策咨询
- 非高价值销售线索

### 2.5 用户真实性（0-10 分）

判断评论用户是否为真实个人用户。

| 分数 | 判定 | 判断标准 |
|:----:|:----:|---------|
| **9-10** | 🟢 高真实性 | 有真实账号（有头像、有作品、有粉丝）、长期活跃、有个人信息 |
| **6-8** | 🟡 中等真实性 | 账号信息部分完整、评论有实质内容 |
| **3-5** | 🟠 低真实性 | 账号信息缺失较多、评论质量一般 |
| **0-2** | 🔴 不可信 | 明确机器人/水军/广告号 → 直接淘汰 |

#### 真实账号判断

| 条件 | 真实信号 |
|------|---------|
| 头像 | 有自定义头像（非默认头像） |
| 作品 | 账号发布过作品 |
| 粉丝 | 有关注者 |
| 账号年龄 | 注册超过 90 天 |

#### 长期活跃判断

| 条件 | 活跃信号 |
|------|---------|
| 近期作品 | 最近 30 天有发布 |
| 评论历史 | 评论内容自然，非机械重复 |
| 互动行为 | 评论有上下文相关性 |

#### 个人信息判断

| 条件 | 个人化信号 |
|------|-----------|
| 个性化昵称 | 非默认格式的昵称 |
| 个人简介 | 填写了有意义的个人简介 |
| 评论内容 | 包含个人场景描述或个性化表达 |

#### 一票否决

以下任一情况出现，用户真实性直接判 **0 分**，评分终止：

| 否决条件 |
|---------|
| 同一评论在 3 个以上视频下复制粘贴 |
| 评论内容为纯广告、引流链接、二维码 |
| 账号 0 粉丝、0 作品、注册不足 30 天 |
| 评论内容为同行试探（提及竞品品牌恶意比较） |

### 2.6 时间价值（0-10 分）

| 分数 | 时间范围 | 处理策略 |
|:----:|---------|---------|
| **9-10** | 最近 7 天评论 | 最高优先，用户记忆新鲜，触达回应率高 |
| **6-8** | 8-30 天评论 | 仍有触达价值，需快速跟进 |
| **3-5** | 31-60 天评论 | 触达价值降低，选择性跟进 |
| **1-2** | 61-90 天评论 | 基本不主动触达 |
| **0** | 超过 90 天 | **历史评论只用于分析，不触达** |

#### 时间加分

| 加分条件 | 加分 |
|----------|:----:|
| 评论在 24 小时内发布 | +2 |
| 评论在视频发布后 1 小时内 | +1 |
| 用户近期在同一竞品下多次评论（≥3 次） | +2 |

> 加分累计不超过维度满分 10 分。

## 三、客户等级划分

### 3.1 四级划分

| 等级 | 总分区间 | 标签 | 定义 |
|:----:|:--------:|------|------|
| **S** | 85-100 | 核心客户 | 高概率近期安装客户 |
| **A** | 70-84 | 优质客户 | 明确兴趣，需要跟进 |
| **B** | 50-69 | 潜在客户 | 有光伏兴趣，可培育 |
| **C** | <50 | 普通用户 | 无明确购买意向 |

### 3.2 等级详解

**S 级（85-100 分）· 高概率近期安装客户：**

| 典型画像 | 处理动作 |
|---------|---------|
| 成都别墅业主，评论说"想装光伏，多少钱，能上门吗" | 即时推送 CRM，24 小时内人工触达 |
| 重庆民宿老板，评论说"我民宿想装，这个月搞定" | 即时推送 CRM，24 小时内人工触达 |
| 贵阳阳光房业主，评论说"阳光房太晒想装光伏，怎么收费" | 即时推送 CRM，24 小时内人工触达 |

**A 级（70-84 分）· 明确兴趣，需要跟进：**

| 典型画像 | 处理动作 |
|---------|---------|
| 成都普通住宅业主，评论"光伏多少钱？效果怎么样" | 48 小时内跟进，解答疑问推进决策 |
| 绵阳自建房业主，评论"我家想装，发电多少" | 48 小时内跟进 |

**B 级（50-69 分）· 潜在客户：**

| 典型画像 | 处理动作 |
|---------|---------|
| 重庆用户评论"先了解一下""收藏了以后再看" | 纳入培育池，定期内容触达 |
| 区域外用户评论"别墅想装，你们那边能做吗" | 标记为跨区域需求 |

**C 级（<50 分）· 普通用户：**

| 典型画像 | 处理动作 |
|---------|---------|
| 评论"好看""点赞"，无任何购买信号 | 不入 CRM 主库，归档备查 |
| 历史评论超过 90 天 | 仅用于分析，不触达 |

### 3.3 等级分布预估

| 等级 | 预估占比 |
|:----:|:--------:|
| S | ~3% |
| A | ~10% |
| B | ~25% |
| C | ~62% |

## 五、AI 评分流程

```
┌─────────────────────────────────────────────────────────┐
│ Step 1：评论输入                                          │
│   来源：04_COMMENT_DATABASE                              │
│   输入：经 COMMENT_ANALYZER_RULE 分类后的 A/B 级评论       │
├─────────────────────────────────────────────────────────┤
│ Step 2：区域识别                                          │
│   ├─ 提取评论文本中的地点关键词                           │
│   ├─ 读取平台 IP 属地信息                                │
│   ├─ 分析用户昵称中的地名                                 │
│   ├─ 对照川渝黔区域分值表                                 │
│   └─ 输出：区域分值（0-20 分）                           │
├─────────────────────────────────────────────────────────┤
│ Step 3：需求识别                                          │
│   ├─ 匹配需求关键词库（40/30/20/10 分四级）              │
│   ├─ 多关键词命中 → 取最高分                             │
│   ├─ 检查一票否决信号（嘲讽/租客/装不起）                │
│   └─ 输出：需求强度分值（0-40 分）                       │
├─────────────────────────────────────────────────────────┤
│ Step 4：房屋判断                                          │
│   ├─ 匹配十大场景关键词库                                 │
│   ├─ 多场景 → 取最高分                                   │
│   ├─ 商业场景特殊处理（民宿/酒店/茶楼/棋牌室/美容院）     │
│   └─ 输出：房屋场景分值（0-20 分）+ 房屋类型标签         │
├─────────────────────────────────────────────────────────┤
│ Step 5：用户真实性验证                                    │
│   ├─ 检查账号质量（头像/粉丝/作品/年龄）                  │
│   ├─ 检查评论内容个性化程度                              │
│   ├─ 一票否决检查                                        │
│   └─ 输出：用户真实性分值（0-10 分）                     │
├─────────────────────────────────────────────────────────┤
│ Step 6：时间价值计算                                      │
│   ├─ 计算评论距今天数                                     │
│   ├─ 计算评论距视频发布时间                               │
│   ├─ 时间加分计算                                        │
│   └─ 输出：时间价值分值（0-10 分）                       │
├─────────────────────────────────────────────────────────┤
│ Step 7：总分计算                                          │
│   ├─ 五维分值累加                                        │
│   ├─ 等级判定：S(85-100) / A(70-84) / B(50-69) / C(<50) │
│   └─ 输出：总分 + 客户等级                               │
├─────────────────────────────────────────────────────────┤
│ Step 8：进入 CRM                                          │
│   ├─ S 级 → 即时推送 05_CUSTOMER_CRM + 通知              │
│   ├─ A 级 → 批量推送 05_CUSTOMER_CRM                     │
│   ├─ B 级 → 纳入培育池                                   │
│   └─ C 级 → 归档备查                                     │
└─────────────────────────────────────────────────────────┘
```

## 六、人工反馈闭环

### 6.1 反馈机制定义

人工跟进结果反向优化 AI 评分。每次销售跟进后，更新 CRM 跟进状态，系统据此评估评分准确性。

### 6.2 跟进状态

| 状态 | 含义 | 对评分的反馈 |
|------|------|------------|
| **已联系** | 销售已首次触达客户 | 验证联系方式有效性 |
| **已回复** | 客户已回复销售消息 | 正向信号：客户有意向 |
| **已加微信** | 客户添加销售微信 | 强正向信号：深度意向 |
| **已测量** | 已上门勘测 | 确认信号：客户真实且有意向 |
| **已成交** | 已完成签约 | 最终验证：评分准确 |
| **无需求** | 客户表示无安装需求 | 负向信号：AI 误判 |

### 6.3 反馈优化规则

| 反馈结果 | 优化动作 |
|----------|---------|
| 连续 3 个 S 级客户最终"无需求" | 调高 S 级阈值 / 检查需求关键词匹配是否过宽 |
| 连续 5 个 B 级客户最终"已成交" | 调低 B 级阈值 / 检查是否漏判高价值客户 |
| 某区域客户大量"已成交" | 提升该区域的分值权重 |
| 某房屋场景客户频繁"无需求" | 降低该场景的分值权重 |

### 6.4 反馈数据格式

```json
{
  "lead_id": "LEAD-20260711-0042",
  "ai_score": 95,
  "ai_grade": "S",
  "follow_up_status": "已成交",
  "actual_value": "别墅 20kW 系统，合同金额 ¥98,000",
  "feedback_date": "2026-07-25",
  "sales_notes": "客户本身就有安装需求，看到评论后立即私信达成联系"
```

# 八、农村光伏客户价值动态修正规则 V1.1

> 农村客户不直接判定低价值。
>
> PV_OS 根据安装意愿、物业条件、付款可能性进行动态评分。

---

## 8.1 农村客户分类

### A类：农村自费安装客户（高价值）

识别信号：

- 自建房
- 宅基地
- 老家房子
- 农村屋顶
- 农村别墅
- 养殖棚
- 农家乐
- 民宿

同时出现：

- 想装
- 准备装
- 多少钱
- 怎么安装
- 能装多少
- 联系方式

判断：

属于真实购买需求。

处理：

正常进入评分模型。

---

### B类：农村潜在客户（培育价值）

关键词：

- 农村能不能装
- 农村适合光伏吗
- 老家可以安装吗
- 回老家看看

判断：

存在兴趣，但购买时间不确定。

处理：

进入 B 级培育池。

---

### C类：免费政策期待客户（低价值）

关键词：

- 免费安装
- 国家免费装
- 政府什么时候安装
- 有没有补贴
- 政策什么时候下来

判断：

主要关注政策，不代表购买。

处理：

降低需求评分。


---

## 8.2 农村客户评分调整

|情况|调整|
|-|-|
|农村自建房 + 明确安装|正常评分|
|农村自建房 + 主动询价|需求分增加|
|农村 + 要联系方式|按S级处理|
|农村 + 只问免费政策|-25分|
|农村 + 只等政府项目|-20分|
|农村 + 只问补贴|-15分|

## 8.3 农村客户最终判断原则

---


农村客户不因“农村”标签自动降低价值。

AI必须结合：

1. 是否存在明确安装需求
2. 是否存在支付能力信号
3. 是否存在房屋/经营场景
4. 是否只是等待免费政策

最终判断：

| 类型 | 判断 | 处理 |
|-|-|-|
| 农村自建房 + 明确想安装 | 真实购买需求 | 正常评分，进入客户池 |
| 农村自建房 + 主动询价 | 高价值农村客户 | 提升优先级 |
| 农村经营场景（民宿、农家乐、养殖、合作社、大面积屋顶） | 商业价值客户 | 按商业客户处理 |
| 农村客户询问安装条件 | 潜在客户 | 保留分析 |
| 农村客户只询问免费安装 | 政策型用户 | 降低评分 |
| 农村客户只等待政府项目 | 非即时成交 | 进入观察池 |
| 农村客户只关注补贴 | 低转化概率 | 降低优先级 |

核心原则：

农村 ≠ 低价值。

真正判断标准：

购买意愿 > 房屋条件 > 区域 > 农村标签

如果农村用户出现：
- 想装
- 报价
- 联系方式
- 上门测量

必须进入正常销售流程。
