# COMMENT_ANALYZER_RULE —— 评论区潜在客户识别算法 V1.0

> PV_OS 评论潜客发现系统核心算法：从竞品评论区中识别城市家庭光伏潜在客户。
>
> 输出对接：`05_CUSTOMER_LEADS/`
>
> 最后更新：2026-07-11

---

## 一、评论分析目标

PV_OS 评论区分析系统聚焦以下两个核心目标：

### 1.1 从竞品视频评论中发现城市家庭光伏潜在客户

- 监控已入库 S/A/B 级竞品账号的评论区
- 从自然语言评论中提取光伏安装需求信号
- 识别出有真实安装意向的个人用户

### 1.2 区分真实需求、兴趣用户、普通互动

| 用户类型 | 定义 | 处理策略 |
|----------|------|---------|
| **真实需求用户** | 已进入决策或评估阶段，可能近期成交 | 即时推送 CRM，S/A 级优先跟进 |
| **兴趣用户** | 对光伏有兴趣但尚未进入决策阶段 | 纳入培育池，定期触达 |
| **普通互动用户** | 仅点赞、表情、无实质光伏讨论 | 仅统计，不入库 |

## 二、评论价值分类体系

### 2.1 S 级：强意向评论

用户明确表达光伏安装意向或主动寻求联系，属于最高优先级。

| 意图类型 | 典型评论示例 |
|----------|------------|
| 我要安装 | "我要安装""准备装一套""想做光伏""最近想装" |
| 询问收费 | "怎么收费""安装费多少""一套下来多少钱" |
| 请求上门 | "能不能上门测量""可以来我家看看吗""能上门吗" |
| 询问本地服务 | "成都哪里可以做""重庆有吗""贵阳有没有" |
| 确认可行性 | "我家能安装吗""这种屋顶可以装吗""老房子能装吗" |
| 直接要联系方式 | "联系方式""电话多少""加个微信""怎么联系" |
| 明确时间意图 | "这个月准备装""年底前想弄好""马上要装修一起做" |

**S 级判定规则：** 命中上述任意 1 条即判定为 S 级。

### 2.2 A 级：明确兴趣评论

用户表现出对光伏的明确兴趣，有具体问题但尚未进入决策阶段。

| 兴趣类型 | 典型评论示例 |
|----------|------------|
| 询价 | "多少钱""价格多少""贵不贵""预算大概多少" |
| 发电效果 | "发电多少""一天能发几度""够不够用" |
| 安全/质量顾虑 | "漏水吗""会不会漏水""安全吗""台风天怕不怕" |
| 产品寿命 | "寿命多久""能用几年""多久换一次" |
| 政策咨询 | "政策怎么样""还有补贴吗""国家补贴多少" |
| 收益计算 | "多久回本""划算吗""能省多少电费" |
| 售后保障 | "售后怎么样""坏了谁修""保修多久" |
| 技术规格 | "多少千瓦合适""需要多大面积""用什么牌子好" |

**A 级判定规则：** 命中上述任意 1 条且不满足 S 级条件，即判定为 A 级。

### 2.3 B 级：潜在兴趣评论

用户表现出对光伏的间接兴趣或场景关联，需求尚未明确但值得培育。

| 兴趣类型 | 典型评论示例 |
|----------|------------|
| 场景描述 | "我家是别墅""我家有个大露台""阳光房太晒了" |
| 观望/收藏 | "先看看""收藏了""以后再说""了解一下" |
| 对比/推荐请求 | "有没有推荐""哪家好""和 XX 比怎么样" |
| 替人询问 | "帮我爸妈问一下""朋友想装" |
| 跨品类关联 | "装修的时候一起弄""和阳光房一起做" |
| 邻居效应 | "邻居装了看着不错""我们小区有人装" |
| 间接需求 | "电费太高了""夏天开空调电费受不了" |

**B 级判定规则：** 命中上述任意 1 条且不满足 S/A 级条件，即判定为 B 级。

### 2.4 C 级：普通互动评论

无光伏相关需求信号，仅为一般性互动。

| 互动类型 | 典型评论示例 |
|----------|------------|
| 赞美/感叹 | "好看""太棒了""厉害""牛" |
| 纯表情/符号 | "👍""😍""[点赞]" |
| @ 好友 | "@张三 来看看" |
| 闲聊/跑题 | 与光伏完全无关的话题 |
| 提问但不相关 | "背景音乐是什么""在哪里拍的" |

**C 级判定规则：** 不命中 S/A/B 级任何条件的评论，归为 C 级。

## 三、用户需求识别规则

对 S/A/B 级评论，逐条分析以下五个需求维度：

### 3.1 是否主动询价

| 判定结果 | 特征 |
|:--------:|------|
| **是** | 评论含"多少钱""价格""费用""报价""收费""预算"等询价关键词 |
| **否** | 评论不涉及价格询问 |

询价 = 强需求信号，S 级标记。

### 3.2 是否咨询施工

| 判定结果 | 特征 |
|:--------:|------|
| **是** | 评论含"安装""施工""上门""测量""勘察""工期"等施工关键词 |
| **否** | 评论不涉及施工咨询 |

咨询施工 = 已进入决策后期，S 级标记。

### 3.3 是否询问地址

| 判定结果 | 特征 |
|:--------:|------|
| **是** | 评论含"地址""在哪""哪里""门店""公司""本地"等位置关键词 |
| **否** | 评论不涉及位置询问 |

询问地址 = 已准备线下接触，S 级标记。

### 3.4 是否描述自己的房屋

| 判定结果 | 特征 |
|:--------:|------|
| **是** | 评论含"我家""我们""别墅""自建房""屋顶""露台""阳光房"等房屋描述 |
| **否** | 评论不涉及房屋描述 |

描述房屋 = 用户已有具体场景想象，A/B 级标记（结合其他信号定级）。

### 3.5 是否询问安装条件

| 判定结果 | 特征 |
|:--------:|------|
| **是** | 评论含"能装吗""可以装吗""适合吗""条件""要求"等可行性询问 |
| **否** | 评论不涉及安装条件 |

询问安装条件 = 正在评估可行性，属于评估期用户。

| 优先级 | 来源 | 提取方式 | 可靠性 |
|:------:|------|---------|:------:|

| 判定结果 | 条件 |
|:--------:|------|
| **川渝黔精确匹配** | 明确识别到川渝黔具体城市/区县 |
| **川渝黔省级匹配** | IP 或资料显示四川/重庆/贵州，但无具体城市 |
| **西南区域** | IP 显示云南/西藏等西南省份 |
| **全国其他** | 明确非川渝黔区域 |
| **未知** | 四个来源均无区域信息 |

## 七、潜客等级划分

### 7.1 四级划分标准

| 等级 | 总分区间 | 含义 | 处理动作 |
|:----:|:--------:|------|---------|
| **S** | ≥80 | 高价值潜客 · 即将成交 | 立即人工触达，24 小时内联系 |
| **A** | 60-79 | 优质潜客 · 明确意向 | 重点跟踪，48 小时内分配销售 |
| **B** | 35-59 | 普通潜客 · 潜在兴趣 | 观察培养，纳入培育池定期触达 |
| **C** | <35 | 非潜客 · 无明确需求 | 归档，不入 CRM 主库 |

### 7.2 各等级处理细则

**S 级处理：**

| 步骤 | 动作 | 时限 |
|:----:|------|:----:|
| 1 | AI 生成客户画像摘要（区域 + 房屋 + 需求一句话） | 即时 |
| 2 | 推送 `05_CUSTOMER_LEADS/leads_master.csv` + 钉钉/企微通知 | 即时 |
| 3 | 人工确认线索真实性 | 1 小时内 |
| 4 | 首次触达客户（评论区回复/私信） | 24 小时内 |
| 5 | 记录触达结果，更新 CRM 状态 | 触达后即时 |

**A 级处理：**

| 步骤 | 动作 | 时限 |
|:----:|------|:----:|
| 1 | 推送 `05_CUSTOMER_LEADS/leads_master.csv` | 每批分析完成后 |
| 2 | 分配销售跟进 | 48 小时内 |
| 3 | 首次触达客户 | 72 小时内 |

**B 级处理：**

| 步骤 | 动作 | 时限 |
|:----:|------|:----:|
| 1 | 推送 `05_CUSTOMER_LEADS/nurture_pool.csv` | 每日汇总 |
| 2 | 定期内容触达（点赞/评论互动建立关系） | 每周 |
| 3 | 60 天内无升级 → 降为 C 级归档 | — |

**C 级处理：**

| 步骤 | 动作 |
|:----:|------|
| 1 | 写入归档日志，不入 CRM 主库 |
| 2 | 保留原始数据备查，180 天后可清理 |

## 十、合规与人工确认机制

### 10.1 数据采集合规

| 原则 | 规则 |
|------|------|
| **公开数据** | 仅采集平台公开可见的评论内容，不绕过任何访问限制 |
| **用户隐私** | 评论文本中的手机号、微信号等隐私信息做脱敏处理后再存储 |
| **平台规则** | 遵守抖音/小红书/快手/视频号的用户协议和社区规范 |
| **不自动化私信** | 不使用自动化工具批量发送私信，人工触达由真实销售执行 |
| **数据留存** | 评论数据仅用于内部客户开发，不外泄或转售 |

### 10.2 AI 自动分析边界

| 允许 AI 自动执行 | 必须人工确认 |
|-----------------|-------------|
| 评论价值分类（S/A/B/C） | S 级线索的最终确认 |
| 五维评分计算 | 首次触达客户的私信/回复内容 |
| 区域/房屋场景自动识别 | 客户手机号等隐私信息的脱敏确认 |
| B/C 级线索的自动归档 | 批量客户数据的对外导出 |
| 分析日志生成 | 评分规则的重大调整 |

### 10.3 人工复核触发条件

以下情况强制人工复核，AI 不可自行处理：

| 触发条件 | 复核内容 |
|----------|---------|
| S 级线索（总分 ≥80） | 确认评论分类准确、区域判断正确、客户非同行试探 |
| 用户真实性评分 ≤3 | 确认是否误判（可能是低活跃真实用户） |
| 评论涉及投诉/纠纷/负面舆情 | 确认是否为竞品恶意攻击或真实投诉 |
| 单日 S 级线索 >10 条 | 检查是否有评分膨胀或关键词过度匹配 |
| 用户 IP 属地为川渝黔但评论内容暗示异地 | 区域信息交叉验证 |

### 10.4 人工确认后的流转

```
人工确认 S 级线索
  ├─ 确认通过 → 状态更新为 pushed → 分配销售 → 记录确认人
  └─ 确认驳回 → 降为 A/B 级 → 记录驳回原因 → 反馈优化规则

人工复核低真实性
  ├─ 恢复为正常 → 保留原评分 → 标记"人工恢复"
  └─ 维持低分 → 归档 C 级 → 记录原因
```
