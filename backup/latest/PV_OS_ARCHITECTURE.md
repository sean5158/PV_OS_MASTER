## 一、系统定位

PV_OS 是一个**以 AI Agent 驱动的光伏行业自动化运营系统**，覆盖从数据采集、客户发现、内容生产到销售转化的全链路。

**核心命题：** 如何用 AI 系统性地发现光伏潜在客户，并通过内容营销实现规模化获客与转化。

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **AI First** | 所有可自动化的环节均由 AI Agent 执行，人工仅做审核与决策 |
| **数据驱动** | 运营决策基于平台数据与客户行为数据，不做拍脑袋判断 |
| **模块解耦** | 采集、分析、生产、CRM 各模块独立运行，通过数据管道连接 |
| **渐进式自动化** | 先跑通关键链路，再逐环节提升自动化率 |
| **可观测** | 每个环节有日志、有指标、可回溯 |

## 四、AI Agent 架构

### 4.1 Agent 清单

| Agent 名称 | 职责 | 输入 | 输出 | 存放位置 |
|------------|------|------|------|---------|
| **Comment Analyzer** | 评论区潜在客户识别 | 平台评论数据 | 潜在客户线索 + 意图标签 | `03_AI_AGENT/agents/` |
| **Customer Scorer** | 客户意向评分与分级 | 客户线索 + 行为数据 | 评分报告 + 跟进优先级 | `03_AI_AGENT/agents/` |
| **Competitor Discovery** | 竞品账号自动发现 | 平台搜索 + 关键词 | 竞品账号列表 + 基础画像 | `03_AI_AGENT/agents/` |
| **Competitor Analyst** | 竞品深度分析 | 竞品账号数据 | 策略分析报告 | `03_AI_AGENT/agents/` |
| **Viral Deconstructor** | 爆款视频拆解 | 高互动内容数据 | 拆解报告 + 可复用模板 | `03_AI_AGENT/agents/` |
| **Content Producer** | AI 内容脚本生成 | 产品参数 + 客群画像 + 爆款模板 | 视频脚本 + 图文文案 | `03_AI_AGENT/agents/` |
| **Lead Follow-up** | 客户跟进提醒与话术 | CRM 客户状态 | 跟进建议 + 话术推荐 | `03_AI_AGENT/agents/` |

### 4.2 Agent 协作关系

```
Competitor Discovery ──→ Competitor Analyst
                                │
                                ▼
Viral Deconstructor ──→ Content Producer ──→ 04_CONTENT/outputs/
        │                       ▲
        │                       │
        └──→ Comment Analyzer ──┘
                    │
                    ▼
              Customer Scorer ──→ Lead Follow-up
                    │                    │
                    └────────────────────┤
                                         ▼
                                 05_CUSTOMER_CRM/
```

### 4.3 Agent 执行模式

| 模式 | 触发方式 | 适用 Agent |
|------|---------|-----------|
| **定时批量** | Cron / 调度器触发 | Comment Analyzer、Competitor Discovery、Customer Scorer |
| **事件驱动** | 新数据到达 / 状态变更 | Lead Follow-up、Content Producer |
| **手动触发** | 人工指令 | Competitor Analyst、Viral Deconstructor |
| **链式触发** | 上游 Agent 完成 | Content Producer（← Viral Deconstructor）、Customer Scorer（← Comment Analyzer） |

## 七、技术栈概览

### 7.1 分层技术选型

| 层级 | 技术选项 | 说明 |
|------|---------|------|
| **数据采集** | Python (requests / playwright) | 平台公开数据采集 |
| **数据存储** | CSV / JSON（轻量阶段）；SQLite / PostgreSQL（扩展阶段） | 渐进式升级 |
| **数据清洗** | Python (pandas) + Shell 脚本 | 批量处理管道 |
| **AI 推理** | OpenAI API / DeepSeek API / 本地模型 | 按 Agent 场景选择 |
| **Prompt 管理** | YAML / Markdown 模板 | 版本可控、可审核 |
| **自动化编排** | Python 脚本 + Cron / systemd timer | 轻量调度 |
| **文件管理** | Git（版本控制） + 目录约定 | 无需额外工具 |
| **日志** | 文本日志 → 12_AI_RUNTIME/logs/ | 标准化格式 |

### 7.2 当前阶段技术策略

- **轻量化优先**：不引入重型框架，能用脚本解决的不用服务
- **文件即数据库**：当前阶段用结构化文件（CSV/JSON/Markdown）替代数据库
- **渐进式升级**：当文件量超过管理阈值时，再迁移到数据库
- **模型可替换**：Agent 定义与模型调用解耦，Prompt 模板独立管理

## 八、安全与合规架构

### 8.1 数据安全分层

| 层级 | 数据类型 | 安全措施 |
|------|---------|---------|
| **公开层** | 平台公开内容、竞品公开数据 | 正常采集，无需额外保护 |
| **内部层** | 客户线索、评分数据、运营策略 | 目录隔离，不对外暴露 |
| **敏感层** | 客户个人信息、合同、财务数据 | 脱敏处理，访问控制 |

### 8.2 合规检查点

```
每次采集前 → 检查平台 robots.txt
每次存储前 → 检查是否包含个人隐私信息
每次发布前 → 人工审核 AI 生成内容
每次分析后 → 确认数据来源可追溯
```

## 九、扩展路线图

| 阶段 | 里程碑 | 关键变化 |
|------|--------|---------|
| **Phase 1（当前）** | 单机文件系统 + 手动触发 | 验证核心链路：采集→识别→CRM |
| **Phase 2** | 定时自动化 + 数据库 | 引入 SQLite，实现无人值守采集 |
| **Phase 3** | 多 Agent 协同 + 仪表盘 | 工作流引擎升级，运营数据可视化 |
| **Phase 4** | 云端部署 + API 化 | 服务化，支持多用户/多渠道 |
